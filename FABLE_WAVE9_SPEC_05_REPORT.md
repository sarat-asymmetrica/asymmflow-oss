Orders-side consumption is the sequenced post-wave step the spec deferred behind B10 — the shared component + `order` mode are in place for it.

Kept `"Delivered"` + real POD `signed_by` and removed the dead `'Signed'` status. Delivery still produces evidence; flagged as a scoped deviation from the literal AC.
5. **B6 backend idempotency is a latent-bug fix, not a posting change.** The guard now keys on an existing `StockMovement` (the real "already applied" evidence) under a row lock — it prevents a double-count of received quantity/inventory. No rounding/tax/posting-order/money math changed; flagged for visibility.
6. **B9c KPI targets corrected.** Revenue/Net Profit repointed from the circular finance/dashboard to Accounting (P&L) so every KPI drills somewhere meaningful.
7. **C4 fixed at the resource, not the symptom.** Stopping the cache cleanup goroutine per test removes the cross-test contamination A7 identified; three full-suite runs were used to confirm (see the C4 result line).
8. **Bindings regenerated centrally** via `wails generate module`; the CRLF-only no-op binding files were reverted so the commit carries only the 5 real-content files (App + CRMService bindings for the 3 new/changed methods, `models.ts` for `DeliveryNoteHeaderInput` + `GRNResponse.is_completed`).
