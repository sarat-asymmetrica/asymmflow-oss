# AsymmFlow Documentation

This tree is organized by *intent* — read `canon/` and `roadmap/` first; everything
under `history/` is an archived record, not a current source of truth.

| Folder | What's in it |
|---|---|
| **`canon/`** | The current architectural law — constitutions, boundary guides, operating principles. Read these first. |
| **`roadmap/`** | Active forward-looking plans. The north star is `roadmap/SOVEREIGN_SUBSTRATE_6_MONTH_ROADMAP.md`. |
| **`active-goals/`** | Live work specs (cashflow evidence, business-memory registry, engine generalization audit, planned conflict resolution). |
| **`status/`** | Module/feature status ledgers and honest "what's really done" audits. |
| **`architecture/`** | Stable reference: target architecture, refactor philosophy, sprint methodology, Butler pipeline spec. ADRs live in `architecture/adr/` (`ADR-001` ratifies the persistence & sync stack). |
| **`ops/`** | Runbooks & checklists: release, backup/restore, secret-config baseline, regression-harness plan. |
| **`modules/`**, **`templates/`** | Module manifests and the manifest template (active contracts). |
| **`compliance/`** | Permission matrix, process/control matrix, resilience & load dossiers (active). |
| **`user-guides/`** | End-user manuals per role (admin, manager, sales, operations, staff). |
| **`design/`** | Dashboard design-system plan. |
| **`marketing/`** | Product catalogue, enterprise briefs, capability profile. |
| **`history/`** | Archived completed work — wave handoffs & progress, goal records, session notes, security fix records, past testing/cleanup audits. Kept for provenance; not current. |

> Root-level docs that stay at the top: `README.md`, `CLAUDE.md`, `AGENTS.md`,
> `SYNTHETIC_IDENTITY.md`, `LICENSE-ROADMAP.md`, `CLA.md`, and the currently-active
> mission handoff (`FABLE_WAVE1_HANDOFF.md`).
