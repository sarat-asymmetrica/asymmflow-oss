# Staff User Guide

Last updated: 2026-04-25

## Role Purpose

Staff users use AsymmFlow for daily work visibility, assigned tasks, team collaboration, and notifications. The production staff role is intentionally conservative: it focuses on Dashboard, Work, and Notifications. Additional read-only screens can be granted by an administrator if the business requires it.

## Dashboard

Open Dashboard after login.

Use it to review:

| Area | Meaning |
|---|---|
| Task summaries | Your workload and overdue items |
| Alerts | System or business items requiring attention |
| Operational cards | High-level business context |
| Finance or sales summaries | Visible only if your role has those permissions |

Dashboard cards are informational. Use Work or the relevant module to update records.

## Work Hub

Open Work from the sidebar.

Tabs:

| Tab | Use |
|---|---|
| My Work | Tasks assigned to you |
| Team Board | Shared task board if your role permits |
| Projects | Project list and project task context |

Create task fields:

| Field | What to enter |
|---|---|
| Task title | Clear action, e.g. `Confirm delivery slot with Gulf Smelting` |
| Priority | Low, Medium, High, or Urgent |
| Due date | Date the task must be completed |
| Project | Related project, if any |
| Assignee | Person responsible |
| Description | Context, blockers, expected outcome |

Task detail fields:

| Field | What to enter |
|---|---|
| Title | Updated task title |
| Priority | Current priority |
| Description | What needs to happen |
| Blocker | Dependency, owner, or next step preventing progress |
| Assignee | Responsible employee |
| Due date | Updated due date |
| Comment | Progress note, blocker update, or completion evidence |

Task actions:

| Action | Result |
|---|---|
| Start | Moves task to in progress |
| Complete | Marks task complete |
| Block | Marks blocked and stores blocker note |
| Unblock | Returns blocked task to open |
| Comment | Adds progress note |
| Delete | Removes task where permitted |

## Projects

Project fields:

| Field | What to enter |
|---|---|
| Project name | Customer/order/internal project name |
| Project type | Category of work |
| Description | What the project is for |
| Member role | Person's role on the project, e.g. Owner, Support, Reviewer |

Use projects when several tasks belong to the same customer, delivery, finance close, or deployment effort.

## Notifications

Open Notifications from the sidebar.

Use notifications to:

| Item | Action |
|---|---|
| Assigned task | Open the task and update status |
| Mention/comment | Review context and reply in the source task/project |
| Approval message | Only act if your role is authorized |
| Delete approval | Admin/authorized users only |

## Good Data Habits

1. Use exact customer, supplier, invoice, order, PO, or DN references in comments.
2. Put decisions in the system rather than only in WhatsApp/email.
3. Mark tasks blocked when waiting for someone else.
4. Mark tasks complete only after the actual work is finished.
5. Do not request admin access for routine task updates; ask for the specific screen or record permission you need.
