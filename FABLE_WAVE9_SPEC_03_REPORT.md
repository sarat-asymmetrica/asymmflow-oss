## Decisions taken (orchestrator, financial-integrity / integration)

1. **Book-bank single-write, not create-then-patch (B1b).** Cross-signals during the wave left the Go `CreateBookBankReconciliation` at a clean 6-arg (carries the confirmed DIT/outstanding-cheque totals in one create) while the frontend still did a 4-arg create + a separate `UpdateBookBankReconciliationAdjustments`. I reconciled to the **single-write 6-arg create** (fewer writes, no partial-failure window) and left `UpdateBookBankReconciliationAdjustments` available as a bound method for editing an existing non-finalized recon's adjustments. Book-bank remains a non-posting proof record — no GL journal.
2. **B4 labeled per real OSS sources, inverting the spec's default wording.** The spec's default ruling ("AccountingScreen = live/real-time; FinancialDashboard = audited/imported") is backwards for OSS: AccountingScreen's statements read the imported Tally staging tables and FinancialDashboard reads live/audited invoice data. "Accurate red beats false green" — I labeled by actual data source and flag the inversion here for owner confirmation of wording. (Constitution Article VII deviation, below.)
3. **C4 conversion lands on `Sent`, not `Draft`.** Draft is in `customerInvoiceClosedWorkflowStatuses` (excluded from aging/VAT), so converting to Draft would silently keep a "final" invoice out of the books. Sent is the first collectible status, matching a normally-sent invoice. Tested.
4. **Fake metrics removed, not relabeled (B6).** Runway (hardcoded `12.5`) and the burn/cash estimates were replaced with real receivables/payables numbers rather than relabeling a fiction as a trader metric — honesty over cosmetic compliance.
5. **DocumentsService delegates for the statement-preview trio.** The B1d preview methods are `func (a *App)` but the frontend imported them (with their existing `ImportBankStatementWithDialog` sibling) from `DocumentsService`; I added the 3 delegates so they bind on DocumentsService for architectural consistency rather than splitting the import.
6. **Bindings regenerated centrally** via `wails generate module` (verified deterministic — empty diff on a clean tree), so no two coders raced on the generated files. The two concurrent-edit hotspots (`app_services.go`, `service_finance.go`) were verified to have composed both coders' changes with no clobber.
