║                         CONFIGURATION                                     ║
╚═══════════════════════════════════════════════════════════════════════════╝

const (
    circuitBreakerThreshold = 3               // Failures to trigger OPEN
    circuitBreakerCooldown  = 5 * time.Minute // Wait before retry
)

Tuning Guide:
  - Threshold: Lower = faster failure detection, higher = more tolerance
  - Cooldown: Shorter = faster recovery, longer = less retries during outage

Default values (3 failures, 5 min) chosen for:
  ✅ Quick detection (3 attempts = ~90s max)
  ✅ Reasonable recovery window (5 min allows ops to fix issues)
  ✅ Minimal false positives (3 consecutive = likely real outage)

"A garden that cannot weather storms is not resilient."

The circuit breaker is like pruning dead branches:
  - Detects when a path (PyMuPDF) is failing
  - Stops wasting energy on that path
  - Routes around the damage (fallback)
  - Checks periodically if the path has healed
  - Restores normal flow when healthy

This is NOT failure. This is RESILIENCE. 🌱
